# v84 — 무엇이 바뀌었나? (쉬운 설명)

## 한 줄 요약
**"위험한 종목은 안 사고, 1-2위 점수가 비슷하면 절반씩 나눠 사도록 바꿨음."**

---

## 무엇이 문제였나?

### 문제 1: 한 종목에 너무 많이 박았다
- 이전 (v83.3): **1위 종목에 90% 박기**
- AEIS처럼 변동성 큰 종목 잡으면 → 잠 못 잠. 하루 -10% 빠지면 portfolio -9%.

### 문제 2: 위험한 종목도 매수 후보에 들어왔다
- 30일 전 최고가 대비 -25% 이상 빠진 종목 (단기 폭락)도 EPS 신호만 좋으면 매수 후보.
- 폭락 중인 종목 잡으면 더 빠질 위험.

---

## v84가 어떻게 바꿨나?

### 변경 1: 1위 종목에 항상 90% 박지 않음 (dynamic weight)

**1위와 2위 점수 차이를 보고 결정**:
- 1위가 **압도적** (점수 차이 15점 이상): **1위에 100% 박기** (2위는 안 삼)
- 1위와 2위가 **비슷** (점수 차이 15점 미만): **1위 50%, 2위 50% 나눠 사기**

**예시 — AEIS (2026-05-28)**:
```
1위 AEIS: 100점
2위 KEYS: 95.10점
점수 차이: 4.90 (15보다 작음)
→ v84 결정: AEIS 50%, KEYS 50%
```
(이전 v83.3 같으면: AEIS 90%, KEYS 10% — AEIS 변동성 부담 컸음)

**예시 — SNDK 압도 시기 (2026-02-27)**:
```
1위 SNDK: 100점
2위 MU: 54.26점
점수 차이: 45.74 (15보다 훨씬 큼)
→ v84 결정: SNDK 100%, MU 안 삼
```
(압도적 1위면 더 집중 = 더 많은 알파)

### 변경 2: 30일 동안 -25% 이상 빠진 종목 매수 후보에서 제외

**dd_30_25 진입 필터** 추가:
- 종목 가격 / 30일 최고가 = -25% 이하면 매수 후보 X
- 단기 폭락 중인 종목 차단

**예시 — AEIS (2026-05-29)**:
```
현재가: $302.18
30일 최고가: $344 (5/14)
drawdown: -12.2%
→ -25% 안 빠짐 → 통과 (계속 매수 후보)
```

만약 AEIS가 추가 -13% 빠지면 (총 -25%) → 자동으로 매수 후보에서 빠짐 (안전판).

## 왜 이렇게 바꿨나? (BT 결과)

73일 데이터 + 100번 다른 시작일 × 3번씩 = **300번 시뮬레이션**:

| 운영 방식 | 평균 수익률 | 이전 대비 |
|-----------|-------------|-----------|
| 이전 (v83.3, 90/10) | +121% | 기준 |
| **v84 (둘 다 적용)** | **+127% / +67%** | **+11.45%p 더 높음** |

(두 가지 환경: SNDK/MU 같은 폭발 종목 포함 / 제외 — 양쪽 모두 우월)

**거의 모든 시뮬에서 v84가 v83.3보다 더 좋음** (양 환경 wins 98/100, 99/100).

---

## v83.3는 왜 그렇게 했나? (사실은 부풀려진 BT였음)

**중요한 발견**: v83.3 (90/10) 채택할 때 사용한 BT가 잘못된 가정.
- v83.3 BT: 매일 비중 다시 조정 (rebalance) — **사용자 의도와 다름**
- 사용자 의도: 한 번 사면 비중 그대로 (entry_fixed)

**같은 v83.3을 entry_fixed로 다시 보니**:
- 부풀려진 BT: 90/10이 80/20보다 +21.45%p 우월 (100% wins)
- **진짜 BT: +1.38%p에 불과** (24/100 wins)

즉 **v83.3 채택 자체가 약한 근거였음**. 이번 v84는 entry_fixed (사용자 의도)로 정직하게 검증한 결과.
