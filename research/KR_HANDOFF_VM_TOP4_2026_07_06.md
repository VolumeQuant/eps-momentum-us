# US → KR 핸드오프: VM top4 재설계 설명 + KR 동형 전략 성과 측정 요청 (2026-07-06)

## 0. 요청 요약
US가 2026-07-05 전략을 재설계해서 실매매 전환했다(아래 상세). KR도 **fwd_PER·EPS 컨센 추정치가 있는 종목만을 유니버스로, "가치 게이트 통과 종목 중 EPS 리비전 상위 top4 동일가중"** 전략이 성립하는지 KR 데이터로 성과 측정해달라. 기존 KR production(E3X6S3 모멘텀)을 바꾸라는 게 아니라 **별도 트랙으로 측정**해보라는 것. ★KR의 "US 핸드오프 직수입 금지, KR 데이터로 재검증" 원칙 그대로 적용 — 구조만 가져가고 임계값은 KR에서 재라.

## 1. US 구시스템이 왜 죽었나 (배경)
- 구시스템 = EPS 리비전 **변화율**(fwd_pe_chg 7/30/60/90일 가중 0.3/0.1/0.1/0.5 + z-score + 3일가중) 순위, 2슬롯 50/50, carryover/에폭/PE_HOLD/EXIT_RANK 패치 누적.
- 치명 결함 3개: ①**절대 밸류 레벨 무시** — PER 83~89짜리 BE, PER 40대 LITE를 1등으로 발송(변화율만 보니까). 유저가 LITE 물려서 실손실. ②**단기창 왜곡** — 실적 계단이 단기창 밖으로 빠지면 추정치가 그대로 높은데도 순위 폭락 → SNDK/MU 같은 winner 강제매도. 이걸 막으려 carryover/PE_HOLD 패치를 덧대다가 ③**장부 붕괴** — env/에폭에 따라 "지금 뭐 보유?"의 답이 4개(BT {MU,SNDK} / 라이브 {AVGO} / 게이트OFF {NVDA} / 유저실제 {LITE,AVGO}). 6월 BT +26.9%인데 유저 실계좌 마이너스 = 시스템이 번 걸 유저가 못 받음.

## 2. 재설계 확정 규칙 (2026-07-05 배포, 유저 실매매 중)
```
유니버스: EPS 컨센(NTM) 있는 미국 대형·중형 ~1,300종목 (매일 수집)
자격:    min_seg ≥ 0        (nc→7d→30d→60d→90d 네 구간 중 하나라도 추정치 하향이면 탈락)
       + 거래대금 30일 ≥ $1B
       + 업종 제외(원자재/엔터/전문소매)
가치게이트: fwd_PER ≤ 30     (price / NTM_EPS)
        + gap ≥ 2.5        (gap = NTM_EPS / TTM실적EPS = 기대성장배수. ★missing=pass:
                            TTM 데이터 없으면 통과 — 컷하면 데이터공백 승자 학살, 전수검증됨)
정렬:    rev90 = 90일 전망상향폭 = (NTM_now − NTM_90d전) / |NTM_90d전| 내림차순
포트:    top4 각 25% 동일가중, 5거래일마다 전체 재선발(고정 캘린더 앵커), carryover/슬롯/이탈룰 없음
방어:    국면 오버레이 별도(HY-OAS 신용스프레드 + S&P200DMA + VIX → 전량현금), 종목 손절/트레일링 없음
```

## 3. 각 부품의 역할 (91일 위상평균 BT, 전부 재스윕 완료)
| 부품 | 역할 | 근거 |
|---|---|---|
| fwd_PER≤30 | **MDD 담당** (수익 아님) | 게이트 없음 +88%/MDD−25 → 30 +115%/−18. 25~35 고원, 30=winner無 세계 최선. 15는 전멸(−5%) |
| gap≥2.5 | 수익+MDD, froth 아닌 진짜 이익폭증만 | 없음 +106/−23 → 2.5 +115/−18. 2.0~3.0 고원. gap↑=winner집중(ex2 붕괴), 2.5=두세계 교차점 |
| rev90 정렬 | **수익 엔진** | rev90 +115 > rev60 +86 > rev30 +76 > rev7 +68 (창 길수록 단조 우위). 90일=실적 사이클 1바퀴와 동기화 — 페이드 시점이 "다음 실적의 심판"과 일치. top4 fwd +5.15% vs 5-10위 +0.13% (예측력 절벽) |
| N=4 | 두 세계 MDD 유일 안정 | winner 유/무 −17/−19. N5는 −15↔−25 뒤집힘, N2(구조)= −27 |
| 동일비중 | 유지 확정 | 기울이기(40/30/20/10, rev90비례, 괴리비례) 전부 winner착시 +MDD 단조악화(−18→−28) |
| R=5 | 고원 내 최소회전 | R4~5 고원, R1 −22p(잔매매), R10+ 감쇠. R15 +120은 위상출렁+ex2악화=노이즈 스파이크 |
| min_seg≥0 | 꺾임 초입 회피 보험 | 0=자연경계. 조임(+0.5)은 즉시 붕괴, 완화(−1~−2) +7p는 강세장 보험제거 착시로 비채택 |

## 4. 판정 방법론 (KR 재현 시 이대로)
- **위상평균**: 리밸 시작 위상 0~4 전부 돌려 평균 (시작일 운 제거)
- **LOWO 누적**: 지배 winner **전부 동시 제거**(US는 −SNDK−MU)한 세계에서도 성립해야 채택. 단일종목 제거는 오해 유발
- **최악 MDD 동반 보고**: 수익만 보고 채택 금지 (비중기울이기·R15가 이 필터에서 죽음)
- **신규 튜닝 0 원칙**: 임계값은 기존에 검증된 숫자만 재사용 (US의 PER30=기존 PE_HOLD, gap2.5=기배포 게이트. in-sample 피크 셀(PER25) 추천했다가 과적합 취약으로 정정한 전례 있음)
- **missing=pass**: 데이터 공백 종목은 게이트 통과시켜라. 구멍을 메워서(전종목 TTM 수집) 컷하면 오히려 죽음(ex2 −16 붕괴) — 전수검증으로 확정된 하중기둥
- **정직한 한계**: US 검증은 91일 단일 강세장(반도체 주도) in-sample. 약세장 방어는 국면 오버레이 몫으로 직교 분리

## 5. KR에서 측정할 것 (요청 본문)
**KR 컨센 커버 유니버스(fnguide, 시총≥1000억 중 커버 ~340+)에서: 자격(유동성/회계필터는 KR 기존 것) → 가치게이트(fwd_PER — KR은 이미 선행PER<20 자격이 유효하다고 독립 검증했으니 그 20부터, 25/30도 스윕) → EPS 리비전 상위 top4 동일가중, 주간 리밸.**

주의·정합 포인트:
1. **KR 기존 결론과의 정렬**: KR도 "자격=선행PER, 비중=기대성장" 구조를 독립 도출했음(forward PER IC 0.147 >> 기대성장 0.041). 이번 제안은 그 위에 **정렬축만 리비전(전망상향폭)으로 교체**한 것. ★KR이 기각한 "가치 sleeve"(기대성장 상위 동일가중, Calmar 0.36~0.54)와 다름 — 그건 **레벨(gap) 정렬**이었고 이번엔 **리비전 정렬 + PER 게이트**. 기각 이력과 혼동 금지.
2. **gap 게이트는 KR서 반대 결론**(gap 자격컷 해로움, `_sleeve_gap_fpe_grid.py`) — KR 버전에서 gap≥2.5는 넣지 말거나 넣어도 참고행으로만. US는 성장프리미엄 시장, KR은 고성장 추격이 고점매수라는 구조 차이 이미 확인됨.
3. **데이터 현실 체크 먼저**: rev90은 컨센 히스토리 90일이 필요. KR 축적 현황 = eps_fusion_daily(6/13~), conviction_fusion_log(6/25~), 리비전 로깅(6/26~), 구 ntm_screening(degrading). **지금 가능한 창(rev10~15)으로 IC부터 측정**(KR은 이미 15일 리비전 IC +0.098을 재본 적 있음, `_kr_revision_test.py`) → 신호 살아있으면 rev30/rev60/rev90은 누적되는 대로 단계 검증. 히스토리 없다고 실제 미래 실적을 프록시로 쓰면 look-ahead 상한임을 명시할 것.
4. **US가 겪은 데이터 함정 2개 점검**: ①파생 지표가 "랭크된 종목만 기록"되는 구멍(US는 거래대금이 top30만 기록돼 MU가 '데이터없음'으로 영구 차단, BT 착시까지 유발 — KR 수집기도 컨센/리비전이 전 커버종목에 기록되는지 확인) ②배포 후 실제 런타임 로그의 WARN 검수(US는 HY-OAS 방어가 배포 후 열흘간 조용히 죽어 있었음).
5. **PER 게이트의 교훈**: US 구시스템(2슬롯 carry)에선 진입 PER캡 BT가 두 번 기각됐는데, top4 재선발 구조로 바꾸니 같은 게이트가 MDD −7%p 개선으로 반전됨. **게이트 유효성은 아키텍처 의존** — KR서 측정할 때도 KR production(3슬롯 carry)이 아니라 topN 재선발 구조 위에서 재야 함.
