> 경위: 사용자 "작업한 내용 면밀히 감사해봐" + "한국 종목 수가 적어 백분위 왜곡 가능" 반박.

## 감사 발견 (근거 데이터)

### 발견 1 — 백분위 분모가 '붕괴 중인 당일 수집분' (사용자 반박 적중)
KR DB 스냅샷(`kr_db_snapshot_2026_07_09.db`, 6/1~7/9 26일) 재분석:
- **수집 붕괴**: KR 유효 유니버스(ntm>0 & n90>0.1) 6/1 **184 → 7/9 69**. 7/2→7/9 단 5일에 멤버십 28% 교체.
- **탈락이 선택적**: 6/1에 있다가 7/9에 사라진 124종목의 6/1 rev90 중앙값 **+11.0** vs 생존 60종목 **+17.1**
  — 차가운 종목부터 수집에서 탈락 → 분모가 점점 뜨거워짐 → 같은 rev90이어도 KR 백분위가 기계적으로 하락.
- **왜곡 정량**: 삼성 7/9 rev90 +63.8%의 백분위 — 당일 분모(69) **84.1%ile** vs 6/1 분모(184) **89.1%ile**
  = 분모 붕괴만으로 **−5.1%p** (이노텍 −5.8%p). top5 경계 여유(문서 주장 ~5%p)와 같은 스케일 →
  "razor-thin 아님" 주장은 분모 아티팩트 위의 결론.
- **"갭 1.2→2.1x 상승추세" = 구성 착시**: 26일 전부 존재한 고정 코호트 39종목의 중앙값은 20.4→22.8로
  평평. 풀샘플 중앙값 상승(12.5→17.0)은 attrition. 단 **KR>US 편향 자체는 진짜**(코호트 갭 2.2~3.1x로
  오히려 더 큼) — 정규화 방향은 유지, 레벨(분모)만 결함.
- 기존 강건성 검증(LOO 69회·부트스트랩 500회)은 "살아남은 69개"에서의 재추출이라 attrition 편향을
  원천적으로 못 잡음 — 강건성 과대평가였음.

### 발견 2 — KR 가치(gap) 게이트 조용한 전멸 (7/9)
- 원장 7/8 블록엔 삼성 gap=4.37 정상, **7/9 블록은 KR 전원 gap 공란**.
- 원인: 7/9 블록은 회사PC schtask가 아니라 **GH Actions 샘플 실행**(`send_unified_real_2026_07_09.py`)이
  기록 — 러너에 fs_dart parquet 없음 → `_kr_ttm_eps` 전건 None → missing=pass로 KR 전원 무검사 통과.
- US는 검사받고 KR은 안 받는 비대칭이 무감지로 발생 + 샘플 실행이 append-only 공식 원장을 오염.

### 발견 3 — bat pull 조용한 실패 = 낡은 코드 실행 구조
- 구 `run_unified_track.bat`: `git pull --ff-only origin master >nul 2>&1` + 원장은 append만(미커밋).
- 원장이 원격에도 커밋되는 순간(7/8 528b437, 7/9 3cdcadc) 로컬 append와 충돌 → pull이 **무음 실패** →
  그때부터 회사PC는 영원히 옛 코드로 실행. CLAUDE.md "git pull 포함이라 당일 수정 자동 반영"은 이 조건에서 거짓.
- 최악 시나리오: 백분위 전환·가드 제거 이전 코드로 구 절대결합 top5 발송.

### 발견 4 — 리밸 시계 3개
- 표시(is_rebal) = US 거래일 그리드(7/2 앵커) / NAV 리플레이 = 로그 실행일 인덱스 i%5(앵커=로그 첫날 7/8)
  / 교체 diff 기준 = `all_days[idx−5]`(로그일 산술). 지시한 매매와 표시한 누적 성과가 다른 날 리밸.
- `cmd_nav`는 같은날 중복 블록 dedup 없음(수동+schtask 이중 실행 시 보유 가중 왜곡).
- diff 기준이 "직전 리밸일의 top5"가 아니라 로그일 산술이라 결측일 생기면 위상 이탈.
