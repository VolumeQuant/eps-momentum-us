# ntm_90d "glitch" 근본원인 — 진단 정정: yfinance 삼성 데이터는 잘못되지 않았다 (2026-07-10)

## 결론 (7/9 NTM90_GLITCH 문서의 진단을 뒤집음)
**삼성전자 ntm_90d의 7/8 +20% 점프는 데이터 오염이 아니라 실제 사건이다.**
90일 전인 **2026-04-08→04-09에 삼성 NTM 컨센서스가 실제로 하루 +20.4% 점프**했다
(4/8경 1분기 잠정실적 발표: 매출 133조·영업이익 57.2조, 영업익 YoY +755% 역대급 서프라이즈
→ 애널리스트 일괄 상향). rolling 90일 윈도우가 그 계단을 지나가면서 기준선이 정당하게
점프한 것 = **base effect이지 glitch가 아님**. 어제 배포한 `_robust_n90_map` 가드는
이 케이스에서 **정상값을 오염으로 오판해 덮어썼다**.

## 증거 (전부 재현 가능)
1. **교차 윈도우 완전 재현**: 6/08의 ntm_60d = 7/08의 ntm_90d = **34748.89972176308
   (소수점 13자리 동일)**. 서로 다른 날, 서로 다른 lookback 컬럼으로 관측한 "4/9 시점
   컨센서스"가 한 달 간격 fetch에서 바이트 동일 — 랜덤 1일 오염이면 불가능.
2. **기저 곡선 복원**: 5개 컬럼(current/7d/30d/60d/90d)의 (date−lookback, value) 산점이
   **하나의 매끈한 단조상승 곡선**(3/3 23.5천 → 7/9 56.9천, 4개월 2.4배)을 그림.
   겹치는 구간 스프레드 ≤1.5%. 계단은 딱 두 곳: **4/8→4/9 +20.4%**(잠정실적),
   5/2→5/3 +13.3%(확정실적/컨콜 후속 상향 추정).
3. **애널리스트 수 안정**(34~35명, 7/1~7/9) — 커버리지 변동 아티팩트 아님.
4. **라이브 yf 정합**(7/10 fetch): 90daysAgo(≈4/11) 0y 33,652/+1y 37,714 = 계단 이후
   레벨과 일치. 0y current 47,588(+41% vs 90d)·+1y 65,560(+74%) — 상향 사이클 실존.
5. **수집 부실일(N=75·69)과 무관** — 삼성 행 자체는 정상 수집·정상 값이었다.

## 진짜 오염의 시그니처는 따로 있다 — 319660.KQ 대조
319660.KQ는 **7/09 하루에 30d/60d/90d 전 컬럼이 동시 점프**(90d +74.6%, ntm_current는
불변). 야후가 이력 시리즈 자체를 재작성(애널 세트 교체/재베이스)한 것. 이건 가드가
잡는 게 맞는(혹은 최소 애매한) 케이스.

| | 삼성 (실제 계단) | 319660 (야후 재작성) |
|---|---|---|
| 점프 위치 | 과거 특정 달력일(4/9)에 고정 | fetch일에 전 컬럼 동시 |
| 교차 윈도우 | 30일 뒤 다른 컬럼서 동일값 재현 | 자기 이력과 불일치 |
| ntm_current | 독립적으로 연속 | 불변인데 이력만 점프 |

→ **판별 규칙**: 오늘 ntm_90d ≈ 30일 전 행의 ntm_60d (또는 60일 전 행의 ntm_30d)이면
실제 사건 = 오버라이드 금지. 불일치면 재작성/오염 = 가드 정당.

## 트레이딩 영향 (긴급)
- 가드가 삼성 rev90을 97.3%로 부풀림. **올바른 rolling rev90 = (56,896−34,749)/34,749
  ≈ +63.7%** → 삼성은 통합 top5 밖(정규화 첫 산출과 동일: 7위권)이 정답.
- **가드는 ~7/13에 자연 소멸**(6일 중앙값이 계단 레벨을 따라잡으면 발동 중지) → 삼성
  rev90이 64%로 "복귀"하며 top5서 기계적 탈락. **지금 삼성을 사면 2~3거래일 뒤 규칙이
  도로 팔라고 함(자기유발 휩쏘)**.
- 미결 안건 "하이닉스 vs 플렉스 5번째"는 해소: 가드 오작동 제거 시
  **top5 = SNDK·MU·HPE·SK하이닉스·FLEX**(둘 다 IN, 삼성 OUT) — 7/9 밤 정규화 첫 산출과 동일.
- 구조적 함의: KR 대형주는 분기마다 잠정실적발 1일 계단이 생김 → **매 계단의 91일
  후마다** rev90 급락 + 가드 3일 오발동 + 해제 = 순위 휩쏘 반복. 가드 수리 필요.

## 조치 (2026-07-10, 사용자 결정 "가드 빼라")
**`_robust_n90_map` 전면 제거** — 함수 + 호출 3곳(us_candidates·kr_candidates·
_universe_rev90). yf 원본 그대로 사용. 잃는 보호 = 사실상 0 (US 6종목=저분모 노이즈로
랭킹 무관, 319660=top5 근처 아님). 검증: compute() top5 = **SNDK·MU·HPE·SK하이닉스·FLEX**
(7/9 밤 정규화 첫 산출과 동일 복원), 삼성 7위(rev90 63.8%, 84.1%ile), 하이닉스 92.8%ile
vs 플렉스 89.8%ile = 동전던지기 해소.
- rev90의 base effect는 **지표 설계로 수용**(90일 지난 뉴스는 잊는 게 모멘텀의 정의,
  US 어닝 계단도 동일 = 전 종목 공평). 평활화하려면 지표 재설계(B안 유사) = 별도 BT 안건.
- 319660형(야후 이력 재작성) 오염이 향후 top5 경계에서 실제 문제 되면, 그때 교차 윈도우
  정합 체크(오늘 ntm_90d ≈ 30일 전 ntm_60d) 기반으로 재설계 — 중앙값 오버라이드 방식은
  실제 계단을 오폭하므로 재사용 금지.

## 교훈
- "90일 전 값은 하루에 +20% 못 뛴다"는 전제가 틀렸다 — **90일 전이 어닝일이면 뛴다.**
- 이상치 가드는 배포 전에 '이상치'의 대안 가설(실제 사건)을 반증했어야 함.
  판별 재료(교차 윈도우 정합)는 DB에 이미 있었다.
